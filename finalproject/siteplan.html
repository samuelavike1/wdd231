Site Name

Reading Tracker Hub — a simple, personal reading companion that helps users discover new books, keep track of what they’re reading, and stay motivated to finish more books.

Why this name? It is clear and descriptive. It immediately tells visitors that the site is centered on tracking reading progress and organizing book recommendations in one hub.

Optional domain availability: readingtrackerhub.me or myreadinghub.site (domain purchase is not required for this class).

Site Purpose

The purpose of Reading Tracker Hub is to provide a focused space where readers can organize their reading life, discover new books to read, and keep track of their progress over time. The site will:

  • Show curated book recommendations: a catalog of books grouped into simple categories such as fiction, non-fiction, self-improvement, and technology.
  • Help users track their reading: a basic reading tracker where users can record whether a book is “To Read”, “In Progress”, or “Finished”, with optional personal notes.
  • Support simple goals: allow users to set a small reading goal (for example, number of books to finish) and see how many they have completed so far.

This scope allows me to use semantic HTML and responsive CSS along with JavaScript features such as fetching data (from JSON or an API), dynamically rendering book cards, storing preferences with localStorage, and interacting with the DOM.

Scenarios

  • “What book should I read next?” A visitor wants to quickly browse recommended books in a specific category (for example, self-improvement or fiction) and view basic details about each book.
  • “Where can I see what I’m currently reading?” A user wants to see a simple list of books they have marked as “In Progress” and update the status when they finish a book.
  • “How many books have I finished this year?” A returning visitor wants to see how many books they have marked as “Finished” compared to their reading goal.

Color Scheme

The color scheme uses a calm, book-friendly palette with a strong primary color and a warm accent. These will be used consistently throughout the final site.

Primary — #1E3A8A
Site logo, main headings, navigation links, and key interactive elements.
Accent — #F59E0B
Buttons (for example, “Add to Tracker”), badges (like “New” or “Finished”), and subtle highlights.

Neutral colors (white backgrounds with slate grays for text and borders) provide good contrast for long-form reading and keep the focus on the book content.

Typography

Headings (Display)

Playfair Display, 700

Used for H1–H2 elements such as page titles (“Recommended Books”, “My Reading Tracker”) to create a bookish, elegant feel that fits a reading-focused site.

Body & UI Text

Inter, 400/600 — used for paragraphs, lists, labels, buttons, and form inputs. It is clean and easy to read, which is important when showing book descriptions and notes.

Testing & Self-Check

  • HTML Validator: Check the site plan and final project pages using the W3C Markup Validator and fix any errors or warnings.
  • Accessibility: Use WAVE and browser DevTools to verify landmarks, form labels, keyboard focus, and color contrast (especially for buttons and links).
  • Best Practices: Confirm there are no JavaScript console errors, that images include descriptive alt text, and that headings follow a logical order.
  • Performance: Keep CSS simple, optimize images for the web, and stay under the 500 kB page weight guideline for each page.
  • SEO basics: Provide descriptive <title> elements and unique meta name="description" text for each page in the final project.
  • Responsive: Test from ~320px up to desktop widths, making sure that navigation, book cards, and tracker components reflow cleanly without horizontal scrolling.
